# Suggestions Hub

This folder captures actionable, community-focused suggestions for TU-VM so contributors can build on prior ideas instead of re-proposing the same concepts.

## What is in this folder

- `historical-suggestions.md`  
  Consolidated baseline of ideas that already exist in project documentation and changelog notes.
- `community-suggestions-framework.md`  
  A practical governance and execution framework for collecting, triaging, implementing, and closing community suggestions.
- `day-to-day-tooling-suggestions.md`  
  Concrete tooling proposals to reduce maintainer and contributor overhead in daily operations.
- `website-community-pages.md`  
  Suggested website markdown page structure and content model for community participation and transparency.

## How to use this folder

1. Start with `historical-suggestions.md` to avoid duplicates.
2. Use `community-suggestions-framework.md` as the process source of truth.
3. Select tooling proposals from `day-to-day-tooling-suggestions.md` based on implementation priority.
4. Publish community-facing content from `website-community-pages.md` in the website/docs surface.

## Decision principles

- Reuse existing project strengths first (`scripts/`, changelog flow, health checks, gateway diagnostics).
- Prefer transparent, auditable processes over ad-hoc decisions.
- Keep implementation small and iterative (ship thin slices with measurable outcomes).
