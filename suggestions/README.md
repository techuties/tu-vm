# Community Suggestions Framework

This folder captures constructive, implementation-ready suggestions for a
community-driven website and contributor workflow.

The goal is to avoid re-inventing ideas by keeping a durable suggestion history,
clear decision records, and actionable implementation templates.

## Files

- `website-historical-suggestions.md`
  - Historical suggestions extracted from existing project docs and changelog
  - Use this first before proposing new work
- `website-community-framework.md`
  - Suggested website information architecture and community lifecycle
- `website-tooling-framework.md`
  - Tooling and automation recommendations to simplify day-to-day operations

## Suggested Process

1. Read historical suggestions to identify existing ideas.
2. Check if a proposal is duplicate, extension, or replacement.
3. Open a new proposal as a markdown update in this folder.
4. Assign status: `proposed`, `accepted`, `in_progress`, `released`, `declined`.
5. Link accepted/released items to changelog entries.

## Suggestion Record Template

```md
## Suggestion: <title>
- Status: proposed
- Owner: <team or person>
- Source: <community, issue, changelog, internal>
- Problem:
- Proposal:
- Reuse Existing Components:
- Risks:
- Success Metrics:
- Decision Notes:
```
