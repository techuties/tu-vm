# Suggestions Framework

This folder stores **historical and active suggestions** for improving the TechUties VM website and community experience.
The goal is to avoid re-inventing work by keeping proposals reusable, comparable, and trackable.

## How to Use This Folder

1. Read this file first.
2. Review existing suggestion files before adding new ideas.
3. Prefer updating an existing suggestion file when the topic already exists.
4. Add a new file only when the proposal is genuinely new.

## Suggestion Lifecycle

Each suggestion should move through:

1. `proposed` -> idea drafted with clear scope
2. `reviewing` -> technical and community feedback requested
3. `approved` -> agreed for implementation
4. `in_progress` -> being implemented
5. `completed` -> merged/deployed
6. `deferred` -> paused for later
7. `rejected` -> not proceeding (with rationale)

## Evaluation Criteria

Use these checks before approval:

- **Community value**: Does it help users and contributors daily?
- **Reuse over reinvention**: Can we use a proven framework/tool instead of custom-building?
- **Operational simplicity**: Is it easy to maintain on a small team?
- **Security and privacy**: Does it preserve the project's secure-by-default posture?
- **Measurable impact**: Are success metrics defined?
