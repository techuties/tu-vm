# Community Suggestions for Website Evolution

This folder captures **constructional, implementation-ready suggestions** for evolving the TU-VM website/dashboard in a way that:

- builds on historical work already present in this repo,
- avoids re-inventing solved parts,
- improves day-to-day operator/developer workflows,
- and supports a sustainable community contribution model.

## Historical signals used (from repo)

These suggestion files intentionally extend patterns already present in the project:

- Tiered operations model (Tier 1 always-on, Tier 2 on-demand)
- Dashboard service controls and status monitoring
- Security hardening (token + allowlist + TLS + audit-oriented flow)
- Operational scripts and “safe update” workflows
- Changelog roadmap hints (quick action profiles, usage analytics, mobile improvements)

## Files

- `website-foundation-and-community.md`  
  Proposal for website architecture, governance, and contribution-friendly structure.

- `website-day-to-day-tooling.md`  
  Practical tooling stack to make daily operations and documentation updates easier.

- `website-feature-suggestions.md`  
  Prioritized feature suggestions for the website/dashboard with acceptance criteria.

## How to use these suggestions

1. Start from `website-feature-suggestions.md` to select quick wins.
2. Use `website-foundation-and-community.md` to align architecture and contribution flow.
3. Implement tooling from `website-day-to-day-tooling.md` to reduce maintenance cost.
