# Community Suggestions Library

This folder stores **constructional, reusable suggestions** for improving the TU-VM ecosystem with a strong community focus.

## Why this exists

To avoid re-inventing the wheel, this directory is the canonical place to:

- capture high-value ideas once,
- describe implementation details in enough depth to execute,
- track tradeoffs and expected impact,
- keep future contributors aligned on priorities.

## How to use this folder

1. **Read existing suggestions first** before proposing a new one.
2. Prefer extending an existing suggestion file when the scope matches.
3. Create a new suggestion file only if the topic is genuinely new.
4. Keep suggestions actionable: include architecture, tooling, rollout, and success criteria.

## Current suggestion set

- `website-community-framework.md`  
  Community architecture for website information flow, governance, and contribution ownership.
- `website-contributor-experience.md`  
  Contributor onboarding, contribution pathways, and recognition systems.
- `website-tooling-automation.md`  
  Tooling, moderation, QA, release hygiene, and day-to-day automation.

## Suggestion quality checklist

Use this quick quality bar for each suggestion:

- clear problem statement and target users,
- implementation framework (not just an idea),
- measurable outcomes and adoption metrics,
- maintenance owner and review cadence,
- migration path from current state.
